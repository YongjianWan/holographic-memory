## 【塞入 ②方案与规格_慢过期.md — 新增独立一块,建议放在 "P1-4 规格" 之后、"P2 已veto" 之前】

### 同义/黑话 lexicon — P1-2/P1-4 的副产派生表（规格定,产出 gated 继承上游）

> 来源:本次对话。解决 FTS5/Jaccard 词面匹配的同义盲区(母红线"无语义召回"的具体疼点),但**不引入任何向量/embedding**。
> 一句话:**同义关系的箭头,黑话部分只有 LLM 读你自己语料才填得出;词频统计能圈出"哪些词是高频黑话",填不出"它的同义词是什么"。** 故让 P1-2/P1-4 那趟已有的 LLM 调用顺手副产同义映射,不新增组件。

**四条规格:**

1. **存哪:独立派生表,非 fact。** 用库里现成的 `semantic_equivalence_groups` / `semantic_equivalence_terms`(本就为此留,当前 0 条)。
   - 与 summary 同待遇:**不进 facts 表、不被 P0 探重、不被 trust 衰减、不参与 RRF 排序竞争、可丢可重建。**
   - 铁律③不咬它:lexicon 是 fact 的派生投影,源料(产它的那些 fact)都在 → 投错重投。删了能重建。

2. **谁产:P1-2 收敛 / P1-4 归纳那趟 LLM 调用的副产物。** 不新增 LLM 调用、不新增组件、不碰写事务、不撞并发补丁①。
   - LLM 细判两条 fact 关系时,顺手回一句"这俩用了不同词指同一概念:X/Y",落 lexicon 表。
   - 这是为什么它非挂在 P1-2/P1-4 上:**黑话同义箭头的唯一可能来源是"LLM 读你自己的语料"**——word2vec 对黑话瞎(训练语料没你的词)、词林对黑话瞎(通用词典没你的词),只有读真实语料的那次 LLM 调用填得出"探重=去重"这种你自造词的桥。

3. **怎么用:查询时同义扩展,低权重兜底。** query 里的词命中 lexicon → 扩展同义词进 FTS5 召回。
   - ⚠️ **扩出来的词给低权重,不与原词在 RRF 平权竞争**——套 §A 线"乘法 boost ±10% 限幅"的审美。否则同义噪声盖过精确命中(同 §3① 怕的"高频/杂义把全库连一团"病,只是发生在词面层)。

4. **严格 gated,继承上游:** P1-4 gated on Gate A;P1-2 生产策略 gated on 四个阈值标定。**lexicon 是它俩的副产物,故自动继承这俩闸——它俩没解禁,lexicon 没原料来源。**
   - ⚠️ **不许为了早点拿 lexicon 把 P1-2/P1-4 提前放开。** 那是拿未验证机制喂新需求,典型 runs ahead。

**唯一现在就能动、不等任何闸的种子来源:** 召回审计手标。跑审计把 miss 捞出来、手标几十条黑话同义对 → 填 lexicon 当初始种子(不依赖 P1-2,是手标)。这批手标还能当 P1-2 解禁后的**验证基准**(LLM 碰撞出的同义对跟手标的对不对得上),并顺带回答"黑话到底有多少"。

> ⚠️ **命名钉死:这不是"边"/"共边"。** 它是"LLM 在合并/归纳时副产的同义映射",进 lexicon 表,**不进任何 edges 表**。叫"共边"会让未来的你/下一个 Claude 误以为在复活 P2,撞坟。

### 改动 2:A 表新增一行(word2vec/GloVe/fastText 预训练词向量)

| 项 | 状态 | 凭什么 | 解冻条件 |
|---|---|---|---|
| **word2vec/GloVe/fastText 预训练词向量** | **悬置(已澄清,未拍板)** | ⚠️ 见下方"射程澄清"——否决它的两道墙(常驻、重资源)经讨论**已倒**,它在红线上**没被干净否决**;但仍卡在"无语义召回"红线内核未拍板 + 工程上未必划算 | (1) Aiden 拍板"无语义召回"红线内核到底是"不常驻"还是"语义判断不依赖外部训练沉淀";(2) 即便红线松开,仍 gated on 召回审计数字 |

**射程澄清(本次记录,①原文不动,此为解读注脚):**

- **❌ "要常驻 embedding 服务" 这条理由,对 word2vec 单次调用作废。** word2vec 查近义可以是一次性调用:进程起→加载文件→算一次余弦→返回→退出。**关机即文件、无启动序列**,跟 Hindsight 被砍的理由(假设机器 24h 活着)**性质相反**。母红线 §0 砍不到它。
- **❌ "需要外部重资源/文件大" 不是有效凭据。** ①§2 那把 Hindsight 筛选刀的刀刃是**常驻**,不是文件体积。一个几个 G 的静态文件关机就是文件,不在刀的射程里。(此前用"外部重资源"拦它是口误,非宪法原文。)
- **⚠️ 由此,A 表第 17 行 "embedding 语义召回" 的凭据"要常驻 embedding 服务"——射程仅覆盖"常驻"的 embedding 用法(如 Hindsight 那路四路检索里的常驻 embedding),不自动覆盖"单次调用"的预训练词向量。** 那一行的否决对常驻用法仍成立,对单次调用不自动成立。
- **✅ 仍然站着的唯一红线问题(未拍板):** "无语义召回(睁眼认)"这条的**真实内核**是哪个——
  - 若内核 = "不常驻" → word2vec 单次调用**合法**,可议进场,代价是检索栈多一路不可复现召回。
  - 若内核 = "Holo 的语义判断不依赖外部训练沉淀、整条检索链可复现" → word2vec 单次调用**仍在门外**,与常不常驻、是不是文件无关。
  - **这条这几轮没有最终拍板。属准则层,Aiden 定。**
- **✅ 即便红线松开,仍未必做的工程理由(故 gated on 审计数字):**
  - 对黑话瞎(训练语料没你的自造词)——黑话盲区它和 HRR 一样补不了,得靠 lexicon 手标。
  - 通用同义词林(cilin)已能覆盖大半(显卡↔GPU 这种词典写死、O(1) 查表、纯红线内)。
  - 会引入一路"召回依据=训练沉淀的向量几何距离、你审计不了为什么"的不可复现召回。
  - 故:**值不值 = 通用同义在 miss 里占多少。** 占大头→才值得议;占小头→词林+lexicon 已够,word2vec 自动出局。**这是个要审计数字才能算的账,不是拍脑袋。**

> **记这条的姿态(给未来的你):** 它不是"被否了"(别想推翻),也不是"要做"(别 runs ahead)。它是"拆到只剩一个你没拍的红线问题 + 一个要审计数字才能算的账"。再想起"诶有个训练好的词向量",翻到这条,看到的就是这个准确状态。
